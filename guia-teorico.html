O que é React?

React é uma biblioteca JavaScript de código aberto para construir interfaces de usuário (UI) ricas e interativas de forma eficiente. Ele permite que você componha UIs complexas a partir de pequenas e isoladas peças de código chamadas "componentes".

UI Declarativa

Com o React, você descreve como sua interface deve se parecer em qualquer estado, e ele cuidará de atualizar o DOM de forma otimizada quando os dados mudarem. Você diz *o que* quer exibir, não *como* fazer isso.

O Coração do React: Componentes

Componentes são como blocos de construção independentes e reutilizáveis. Eles encapsulam sua própria lógica e aparência, permitindo que você construa UIs complexas a partir de peças menores.

Visualizando a Composição

Pense em uma página web como um conjunto de componentes aninhados. Esta abordagem torna o código mais fácil de entender, manter e reutilizar.

JSX - JavaScript e XML

JSX é uma extensão de sintaxe que permite escrever marcação semelhante a HTML diretamente em seus arquivos JavaScript. Embora não seja obrigatório, é a forma recomendada de descrever a UI no React.

HTML Tradicional

<button class="btn">Clique Aqui</button>

JSX Equivalente

<button className="btn">Clique Aqui</button>
Atenção: JSX não é HTML! Atributos como `class` se tornam `className` para evitar conflitos com palavras-chave do JavaScript.

State e Props

State e Props são os dois tipos de "dados" que controlam o que um componente renderiza. Entender a diferença é fundamental.

Props (Propriedades)

São usadas para passar dados de um componente pai para um filho. As props são somente leitura; um componente filho nunca deve modificar as props que recebe.

State (Estado)

É um conjunto de dados gerenciado internamente por um componente. Quando o estado de um componente muda, o React o renderiza novamente para refletir essas mudanças. É o que torna a UI interativa.

O DOM Virtual

O DOM Virtual (VDOM) é o segredo por trás da alta performance do React. É uma representação do DOM real do navegador, mantida em memória.

Processo de Atualização Eficiente

  1. Mudança de Estado: Um evento (como um clique) atualiza o estado de um componente.
  2. Renderização Virtual: React renderiza uma nova árvore do VDOM em memória.
  3. Diferenciação (Diffing): React compara a nova árvore do VDOM com a anterior e identifica exatamente o que mudou.
  4. Atualização Real: Apenas as mudanças calculadas são aplicadas ao DOM real do navegador. Isso minimiza as manipulações diretas do DOM, que são lentas.

Guia Prático: Olá Mundo

Vamos criar nossa primeira aplicação "Olá Mundo" de três formas diferentes. Isso ajudará a entender as ferramentas mais comuns do ecossistema.

Comparativo de Ferramentas

Entenda as principais diferenças, vantagens e desvantagens entre Create React App, Vite e Next.js para fazer a escolha certa para o seu projeto.

Create React App

Abordagem: Empacota toda a aplicação com Webpack antes de iniciar.

Vantagens

  • Simples (zero config)
  • Ecossistema maduro

Desvantagens

  • Lento para iniciar/recarregar
  • Considerado legado

Vite

Abordagem: Usa módulos ES nativos para um servidor de desenvolvimento ultra-rápido.

Vantagens

  • Extremamente rápido
  • Configuração moderna
  • Ideal para SPAs

Desvantagens

  • Ecossistema de plugins mais novo

Next.js

Abordagem: Framework completo com foco em SSR e SSG para performance e SEO.

Vantagens

  • Ótimo para SEO
  • Performance de carregamento
  • Roteamento integrado

Desvantagens

  • Mais complexo e opinativo
